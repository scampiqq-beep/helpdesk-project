# Step 9 — audit/timeline и доменные справочники

На этом шаге добавлен доменный слой `helpdesk_app/domain/ticketing.py` и новый `AuditService`.

Что сделано:
- централизованы статусы, приоритеты и причины закрытия;
- HistoryService начал собирать нормализованный timeline с подписями событий;
- TicketDetailService теперь готовит `timeline`, `timeline_count`, `normalized_status`, `normalized_priority`;
- admin audit page больше не пустая: она собирает журнал из `TicketHistory` и `TicketMessage` без изменения схемы БД.

Почему это безопасно:
- отдельная таблица аудита пока не вводится, значит нет риска сломать текущую БД;
- маршруты и шаблоны не менялись радикально;
- новый audit layer работает как bridge-слой поверх существующих данных.

Следующий шаг: вынос доменных справочников для статусов/приоритетов/причин закрытия в UI и постепенная зачистка legacy helper'ов.
